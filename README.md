# 메이플 타이머 - 미디어 자동 정지/재생 (크롬 확장 프로그램)

MesoTimer(타이머 HTML 페이지)의 **알람이 울리는 순간** 다른 탭(또는 사용자가 지정한 탭)에서
재생 중인 영상(유튜브, 넷플릭스, 트위치 등 `<video>`/`<audio>` 태그를 쓰는 모든 사이트)을
자동으로 **일시정지**하고, 타이머가 **리셋되는 순간**(반복 알림 후 자동 재시작, 마우스 휠 보정,
더블클릭 리셋 등) 그 탭들을 다시 **재생**시켜 줍니다.

브라우저 정책상 웹페이지(HTML)만으로는 다른 탭을 제어할 수 없기 때문에,
이 기능은 반드시 크롬 확장 프로그램 형태로 설치해야 동작합니다.

크롬뿐 아니라 엣지, 웨일 등 크로미움 기반 브라우저에서는 폴더 그대로 로드하면
동일하게 동작합니다(별도 분기 처리 필요 없음).

> **⚠️ 5차 개선 업데이트: `index.html`, `background.js`, `popup.html`, `popup.js`, `manifest.json`을 교체해야 합니다.**
> 이번 개선으로 알림 클릭 시 휠 재시작 동작, 팝업의 "정지된 탭 목록", 스누즈(다음
> 알람 한 번 건너뛰기) 기능이 추가됐습니다.
> - `index.html`(타이머 페이지)의 알림 클릭 동작이 바뀌었으니, 깃허브 페이지에
>   올려둔 타이머 HTML을 이 폴더의 `index.html`로 교체·재배포해주세요.
> - `background.js`, `popup.html`, `popup.js`, `manifest.json`(버전만 2.2→2.3)이
>   바뀌었으니, 확장 프로그램 폴더 전체를 이 버전으로 교체한 뒤
>   `chrome://extensions`에서 확장 프로그램을 새로고침(리로드)해주세요.
>   (`watcher.js`는 이번에 바뀐 것이 없지만, 폴더를 통째로 교체하면 함께 다시
>   로드해도 무방합니다.)
> 자세한 변경 내용은 하단 "작업 기록" 섹션을 참고하세요.

## 설치 방법 (최초 1회, 5분 안 걸림)

1. 이 폴더(`maple-timer-auto-pause`) 전체를 컴퓨터의 원하는 위치에 저장합니다.
   (압축파일로 받으셨다면 먼저 압축을 풀어주세요. 이 폴더를 나중에 지우거나 옮기면
   확장 프로그램이 사라지니, `문서` 폴더 등 고정된 위치에 두는 걸 추천합니다.)
2. 크롬 주소창에 `chrome://extensions` 를 입력하고 이동합니다.
3. 화면 오른쪽 위에 있는 **"개발자 모드"** 토글을 켭니다.
4. 왼쪽 위 **"압축해제된 확장 프로그램을 로드합니다"** 버튼을 클릭합니다.
5. 방금 저장한 `maple-timer-auto-pause` 폴더를 선택합니다.
6. 목록에 "메이플 타이머 - 미디어 자동 정지/재생"이 뜨면 설치 완료입니다.

설치 후에는 크롬 툴바 오른쪽 위 퍼즐 아이콘(확장 프로그램)을 눌러
이 확장 프로그램을 고정해두면 켜짐/꺼짐 상태를 바로 확인할 수 있습니다.

## 사용 방법

1. 평소처럼 타이머 HTML 페이지를 한 탭에서 켜두고, 다른 탭에서 유튜브/넷플릭스 등을 재생합니다.
2. 타이머 알람이 울리는 순간(화면이 빨갛게 경고 상태로 바뀌는 순간), 대상 탭의 영상이
   자동으로 일시정지됩니다.
3. 타이머가 리셋되는 순간(반복 알림 지속시간이 끝나고 자동으로 다시 시작될 때, 마우스 휠로
   보정할 때, 더블클릭으로 초기화할 때) **확장 프로그램이 직접 정지시켰던 영상만** 자동으로
   다시 재생됩니다. 사용자가 스스로 일시정지한 영상은 건드리지 않습니다.

끄고 싶을 때는 확장 프로그램 아이콘을 눌러 체크박스를 해제하면 됩니다.

### 특정 탭만 제어하고 싶을 때 (선택 사항)

기본값은 **열려 있는 모든 탭**이 대상입니다. 유튜브에서 노래를 들으면서 동시에 다른
탭에서 방송을 보는 등, 미디어 탭이 여러 개라 특정 탭만 제어하고 싶다면:

1. 확장 프로그램 아이콘을 눌러 팝업을 엽니다.
2. 탭을 추가하는 방법은 두 가지입니다.
   - **지금 보고 있는 탭**: 팝업 맨 위 "현재 탭 빠르게 추가"에서 바로 **도메인** 또는
     **제목** 버튼을 누르면, 목록을 뒤질 필요 없이 그 탭이 즉시 추가됩니다.
   - **다른 열린 탭**: "탭 추가" 목록에서 원하는 탭 옆의 **도메인** 또는 **제목** 버튼을
     누릅니다.
   - **도메인**: 애니메이션/드라마처럼 에피소드마다 탭 제목이 바뀌는 사이트에 적합합니다.
   - **제목**: 유튜브처럼 같은 사이트 안에 여러 용도의 탭(방송 탭 + 음악 탭 등)이
     동시에 열릴 수 있는 경우에 적합합니다. (재생 중인 곡이 바뀌면 탭 제목도 바뀌므로
     재연결이 끊길 수 있지만, 보통 사냥 시간이 곡 하나보다 훨씬 길어 큰 문제가 되지 않습니다.)
3. 선택한 탭은 목록에 표시되며, 삭제 버튼으로 언제든 뺄 수 있습니다.
4. **선택된 탭을 모두 지우면(아무것도 선택 안 하면) 다시 기본값인 "전체 탭 대상"으로
   돌아갑니다.**

선택 정보는 `chrome.storage.session`에 저장되어 **브라우저(크롬)를 완전히 종료하기 전까지만
유지**되고, 브라우저를 껐다 켜면 초기화됩니다.

### 알람 시 브라우저 알림 받기 (선택 사항)

다른 창을 보고 있거나(멀티 모니터 등) 탭을 잠깐 벗어나 있어도 알람을 놓치지 않도록,
타이머 페이지 자체에서 OS 알림을 띄울 수 있습니다.

1. 타이머 페이지 우측 상단 메뉴(⚙) → **SYSTEM** 항목의 **브라우저 알림** 체크박스를 켭니다.
2. 브라우저가 알림 권한을 물어보면 **허용**을 눌러주세요. (이미 차단한 적이 있다면
   체크박스가 다시 꺼지고 안내 문구가 뜹니다. 이 경우 주소창 왼쪽 자물쇠 아이콘 등에서
   이 사이트의 알림 권한을 직접 허용으로 바꾼 뒤 다시 켜주세요.)
3. 알림은 **타이머 탭(또는 PIP 창)에 포커스가 없을 때만** 뜹니다. 이미 화면을 보고 있는
   상태에서 빨간 경고 애니메이션과 소리에 더해 알림까지 중복으로 뜨지 않도록 하기 위함입니다.

이 기능은 `chrome.*` 확장 프로그램 API가 아니라 표준 **Web Notification API**를
사용합니다. 그래서 확장 프로그램 설치 여부와 무관하게, 타이머 페이지(`index.html`)를
연 브라우저가 크롬이든 엣지·웨일 등 크로미움 계열이든, 파이어폭스나 사파리든 동일하게
동작합니다.

체크박스 바로 옆의 **"테스트 알림 보내기"** 버튼을 누르면, 지금 탭을 보고 있는
상태여도(포커스 체크를 건너뛰고) 알림을 하나 바로 띄워 정상 동작을 확인할 수
있습니다. 권한을 아직 켜지 않은 상태에서 누르면 "먼저 브라우저 알림을 켜주세요"
안내가 뜹니다.

반복(loop) 모드가 켜져 있을 때는 알림 본문에 지금까지 쌓인 **LOOT 수**가 함께
표시됩니다(예: "설정한 시간이 되었습니다. (LOOT 12)"). 반복 모드가 꺼져 있으면
기존처럼 고정 문구만 표시됩니다.

알림(테스트 알림 제외)을 **클릭**하면 타이머 페이지에서 마우스 휠을 굴렸을 때와
완전히 똑같이 동작합니다 — LOOT가 하나 적립되고, 삑 소리가 나고, 타이머가 바로
재시작됩니다. 알림 안에 별도의 "지금 리셋" 버튼을 두는 대신, 알림 자체를 누르는
것이 곧 빠른 재시작이라고 생각하시면 됩니다.

### 확장 아이콘 배지로 정지된 탭 수 확인하기

팝업을 열지 않아도 툴바의 확장 프로그램 아이콘에 지금 몇 개의 탭이 "정지된
상태(알람 때문에 확장 프로그램이 멈춰두고 재생을 기다리는 중)"인지 빨간 숫자
배지로 바로 표시됩니다. 리셋이 발생해 해당 탭들이 다시 재생되면 배지는
자동으로 사라집니다. 확장 프로그램 자체를 껐을 때도 배지는 비워집니다.

팝업을 열면 배지 숫자뿐 아니라 **"지금 정지된 탭"** 목록에서 정확히 어떤 탭들이
정지되어 있는지(파비콘·제목·도메인)도 확인할 수 있습니다. 정지시킨 뒤 탭이
닫혔다면 "(닫힌 탭)"으로 표시됩니다.

### 스누즈 — 다음 알람 한 번 건너뛰기

확장 프로그램을 완전히 끄지 않고도, 팝업의 **"다음 알람 한 번 건너뛰기(스누즈)"**
체크박스를 켜두면 바로 다음 알람에서만 미디어를 정지시키지 않습니다. 한 번
사용되면 자동으로 다시 꺼지는 1회성 기능이라, 따로 꺼주지 않아도 그 다음
알람부터는 평소처럼 동작합니다. 잠깐 다른 걸 보고 싶을 때 유용합니다.
(참고: 이 값은 브라우저를 껐다 켜면 초기화됩니다.)

## 동작 원리 (참고)

- 타이머 페이지에 `#timerDisplay` 요소가 있는지 확인해서, "이 페이지가 MesoTimer인지"
  자동으로 판별합니다. (다른 사이트에서는 아무 동작도 하지 않습니다.)
- **정지**: 타이머 알람이 울리면 `<body>` 태그에 `alarm-state` 클래스가 추가되는 것을
  감지해서 확장 프로그램에 신호를 보내고, 대상 탭에서 재생 중인 `<video>`, `<audio>`를
  찾아 `pause()`만 호출합니다. 이때 정지시킨 탭 목록을 기억해둡니다.
- **재생**: 타이머가 리셋될 때마다(자동 loop 재시작, 휠 보정, 더블클릭 등) `<body>` 태그의
  `data-reset-seq` 값이 바뀌는 것을 감지해서 확장 프로그램에 신호를 보내고, **방금 위에서
  기억해둔("확장 프로그램이 직접 정지시킨") 탭에만** `play()`를 호출합니다. 사용자가 직접
  멈춘 영상이나, 애초에 정지시킨 적 없는 탭은 건드리지 않습니다.
- 영상을 끄거나, 페이지를 닫거나, 그 외 다른 조작은 하지 않습니다.

## PIP 모드 관련 수정 사항 (index.html v5.3 → PIP-FIX)

확장 프로그램 자체(`background.js`, `watcher.js`, `manifest.json`)는 이번에
**바뀐 것이 없습니다.** 문제의 원인은 확장 프로그램이 아니라, 타이머 HTML이
PIP(화면 속 화면) 창을 여는 방식에 있었습니다. 원인과 수정 내용은 아래와 같습니다.

### 1) PIP 모드에서 미디어 자동 정지가 안 되던 문제
- 원인: PIP 창을 열면 타이머 화면(`#app-container`)이 원래 탭의 문서에서
  **PIP 창의 별도 문서로 통째로 이동**합니다. `watcher.js`는 확장 프로그램 구조상
  PIP 창(진짜 탭이 아님)에는 주입될 수 없고, 원래 탭의 `<body>`만 계속 감시하고
  있었습니다. 그런데 알람이 울릴 때 `alarm-state` 클래스는 PIP 창 쪽 `<body>`에만
  붙고 있어서, `watcher.js`가 이를 영영 감지하지 못했습니다.
- 수정: 알람 상태가 바뀔 때마다 PIP 창의 `<body>`와 **원래 탭의 `<body>`에 항상
  동시에** `alarm-state` 클래스를 반영하도록(`setAlarmClass()` 헬퍼) 고쳤습니다.
  이제 PIP 모드에서도 `watcher.js`가 정상적으로 알람을 감지해서 확장 프로그램에
  신호를 보냅니다.

### 2) PIP 모드에서 타이머가 0초 이후 숫자가 계속 늘어나던 문제
- 원인: 확장 프로그램과는 무관하고, PIP 창의 조작부(dock)가 좁은 창 너비에 맞춰
  자동으로 축소되면서(`autoFitDock`) 버튼들이 매우 작아지는 것이 원인으로 보입니다.
  이 축소된 상태에서 "카운트 방향" 버튼(`btnMode`, ▽/△ 아이콘)을 실수로 눌러
  "남은 시간(카운트다운)" 모드가 "경과 시간(카운트업)" 모드로 바뀌면, 목표 시간에
  도달할 때까지 0부터 계속 올라가다가 다시 0으로 리셋되는 톱니 모양 패턴이 반복돼서
  "무한히 증가"하는 것처럼 보입니다.
- 수정:
  - PIP 창의 조작부가 일정 크기 이하로는 줄어들지 않도록 최소 배율을 적용해서
    실수로 누르기 어렵게 했습니다.
  - PIP 창의 기본 크기를 살짝 키웠습니다(320×420 → 360×460).
  - "경과 시간(카운트업)" 모드로 바뀌면 카운트 방향 버튼이 빨갛게 깜빡이도록 해서,
    실수로 눌렸을 때 바로 눈에 띄고 즉시 다시 눌러 되돌릴 수 있게 했습니다.
  - 만약 카운트다운 도중 숫자가 다시 늘어나는 게 보이면, 카운트 방향 버튼이
    빨갛게 깜빡이고 있는지 확인하고 한 번 더 눌러 원래대로(▽ 다운 모드) 돌려주세요.

### 3) (추가 안전장치) PIP 중 원래 탭 오조작 방지
- PIP를 열면 원래 탭의 화면은 비어 있지만, 클릭/더블클릭/휠 이벤트가 여전히
  연결되어 있어서 실수로 그 탭을 클릭하면 타이머가 시작/정지/초기화될 수 있었습니다.
  PIP가 열려 있는 동안에는 원래 탭에서의 클릭이 무시되도록 막았습니다.

**적용 방법:** 깃허브 페이지에 올려둔 타이머 HTML을 이 폴더의 `index.html` 내용으로
교체하고 다시 배포(커밋/푸시)하면 됩니다. 확장 프로그램 폴더는 다시 로드할 필요
없습니다(확장 프로그램 코드는 변경되지 않았습니다).

## 권한 안내

- `tabs`, `scripting`: 대상 탭에 "일시정지"/"재생" 명령만 전달하기 위해 필요합니다.
- `storage`: 켜짐/꺼짐 설정(`storage.local`)과 선택한 탭 목록·정지시킨 탭 추적
  (`storage.session`, 브라우저를 끄기 전까지만 유지)을 저장하기 위해 필요합니다.
- `<all_urls>`: 어떤 사이트에서 영상을 보고 있어도 동작하게 하기 위한 권한입니다.
  이 확장 프로그램은 영상 재생/정지 외에 어떤 데이터도 읽거나 전송하지 않습니다.

## 주의사항

- 크롬 웹스토어에 정식 등록된 확장 프로그램이 아니라 "개발자 모드"로 직접 로드하는
  방식이라, 크롬 업데이트 시 가끔 "개발자 모드 확장 프로그램" 경고 배너가 보일 수
  있습니다. 정상 동작에는 문제 없습니다.
- 폴더를 이동/삭제하면 확장 프로그램도 함께 사라지니 주의하세요.
