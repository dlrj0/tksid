# 단축키로 타이머 리셋 — 구현 가이드

> 이 문서는 "다음 Claude"가 이어서 작업할 수 있도록 방향을 정리한 설계 문서입니다.
> 코드 작업을 진행하면서 내용이 바뀌면(설계 변경, 제약사항 발견, 결정 번복 등)
> **이 문서도 함께 최신화**해주세요. 특히 "열린 결정사항"과 "체크리스트"는
> 작업 중 실시간으로 갱신하는 걸 권장합니다.
## 0. 필수 확인사항
- 메이플 정책위반은 절대 안됨.
- 기존의 기능은 유지하면서 코드 작성, 필요하면 더 나은 방향으로 기존의 기능을 개선.
- 재생/정지하는 미디어 탭을 확장프로그램에서 지정을 하는것처럼 단축키를 인식할 창을 확장프로그램에서 선택할 수 있도록 구현(이부분은 현재 열려있는 창의 이름만 인식해도됨. 메이플만 알면되기 때문에 alt+tab실행시 각창의 프로그램 이름이나 헤더 제목같이 나타나는 값으로 인식, 알트탭과 작업표시줄에서 프로그램에 커서를갖다대면 같은 제목이 나타남(ex- 크롬창마다 헤더가 다르고 메이플은 프로그램이름이 나타남. 확장프로그램으로 탭을구분하는것은 조작할 미디어를 정하기위해 자세한 지정이 필요하지만 단축키인식할 창은 자세한 구분이 필요없음))
## 1. 배경 / 목표

- 사이트(MesoTimer)에서 설치기 등 쿨타임을 스크롤로 리셋해 지속시간과 싱크를 맞추며 사용 중.
- 문제: 사냥 중 마우스로 손을 옮겨야 해서 불편함 → **키보드 단축키로 리셋**하고 싶음.
- 핵심 제약: 사용자가 실제로 보고 있는 창은 **메이플 게임 창**이지, 브라우저(타이머 탭)가 아님.
  즉 단축키는 **브라우저가 포커스를 잃은 상태에서도 동작**해야 함.
- 지난 대화에서 이미 "게임에 입력을 주입하지 않고 감지만 하는 구조는 매크로/핵 정의에 해당하지
  않는다"는 정책 판단을 했음. 이번 기능도 동일한 성격(감지만 함, 게임에 아무 것도 넣지 않음)이라
  그 판단이 그대로 적용됨.

## 2. 핵심 결론 (기술 재검증 완료 — 방향이 바뀐 부분)

이전 대화에서는 "일반 확장프로그램은 전역 단축키를 못 잡으니 네이티브 헬퍼 프로그램이
필요할 것"이라고 안내했었는데, **이 부분은 정정합니다.**

Chrome 확장프로그램의 `chrome.commands` API는 커맨드를 `"global": true`로 등록하면
**브라우저가 포커스를 잃은 상태(= 메이플이 포그라운드인 상태)에서도 단축키가 동작**합니다.
별도의 네이티브 프로그램이나 로컬 서버 없이 확장프로그램만으로 해결 가능합니다.

- 근거: Chrome 공식 문서(`chrome.commands`) — "Beginning in Chrome 35, extension
  developers can optionally mark a command as global. Global commands also work
  while Chrome does not have focus."
- 단, 개발자가 manifest에 지정할 수 있는 `suggested_key`는 **global 커맨드에 한해
  `Ctrl+Shift+[0-9]` 조합으로 제한**됨 (다른 앱 단축키를 가로채는 위험을 줄이기 위한
  Chrome 자체 정책). 예: `Ctrl+Shift+9`.
- 사용자는 `chrome://extensions/shortcuts` 화면에서 이 기본값을 다른 조합으로
  **수동으로 재지정**할 수 있음. 하지만 확장프로그램 코드가 임의의 키 조합을
  프로그래밍적으로 등록하는 것은 불가능함 (아래 "제약사항" 참고).
- ChromeOS는 global 커맨드를 지원하지 않음 (참고만, 이 프로젝트엔 큰 영향 없을 가능성 높음).

## 3. 왜 스크롤 대신 이 방식인가 (대안 비교, 참고용)

| 방식 | 메이플 포커스 상태에서 동작 | 손 이동 필요 | 추가 프로그램 필요 |
|---|---|---|---|
| 마우스 스크롤 (기존) | X (타이머 탭 포커스 필요) | O (마우스로) | 없음 |
| 페이지 내 키보드 리스너 (index.html에 이미 있음, KeyR 등) | X (타이머 탭 포커스 필요) | O (탭 전환 필요) | 없음 |
| **`chrome.commands` global 단축키 (이번 목표)** | **O** | **X** | **없음** |
| OS 레벨 저수준 후킹 네이티브 앱 | O | X | 있음 (불필요하게 무거움 + 안티치트 감지 카테고리에 걸릴 여지) |

→ `chrome.commands` global이 가장 가볍고, 별도 상주 프로그램이 없어서 "메이플과 함께
실행되는 비정상 프로그램"으로 오인될 여지도 최소화됨. **이 방식으로 확정.**

## 4. 제약사항 (구현 전에 반드시 인지할 것)

1. **팝업 안에 "키를 눌러서 직접 캡처하는" 커스텀 단축키 설정 UI는 만들 수 없음.**
   `chrome.commands`는 실제 키 바인딩을 JS로 `set()`하는 API가 없음. 바인딩 변경은
   오직 브라우저 자체 화면(`chrome://extensions/shortcuts`)에서만 가능함.
   → 팝업이 할 수 있는 일은 "현재 단축키가 뭔지 보여주기"와 "설정 화면으로 이동하는
   버튼"까지임. 사용자가 "팝업창에서 단축키 설정 UI"를 요청했지만, 실제 키 입력 캡처는
   브라우저 네이티브 화면에서 이뤄진다는 걸 UI 문구로 명확히 안내해야 함(오해 방지).
2. `chrome.commands.openShortcutSettings()`로 설정 화면을 바로 열 수 있음(비교적 최신 API).
   next Claude가 구현 시점에 사용 중인 Chrome 버전에서 지원되는지 재확인 필요.
   미지원 시 fallback: `chrome://extensions/shortcuts` 안내 텍스트 + 복사 버튼.
3. 확장프로그램 하나가 등록 가능한 suggested 단축키는 최대 4개. 지금은 리셋 1개만 필요.
4. **메이플 클라이언트 자체 단축키와 충돌 여부는 사용자가 직접 확인해야 함.**
   `Ctrl+Shift+숫자` 조합이 메이플 기본 단축키와 겹치지 않는지 실제 게임에서 테스트 필요
   (겹치면 게임 쪽 동작이 우선될 수 있고, 반대로 확장프로그램이 게임 단축키를 먹어버릴
   수도 있음 — 실측 필요).

## 5. 구현 계획

### 5-1. `manifest.json`
`commands` 키 추가:

```json
"commands": {
  "reset-timer": {
    "suggested_key": { "default": "Ctrl+Shift+9" },
    "description": "메이플 타이머 리셋 (전역 단축키)",
    "global": true
  }
}
```
- 별도 permission 불필요 (commands는 manifest key만으로 동작).

### 5-2. `background.js`
- `chrome.commands.onCommand.addListener((command) => { ... })` 추가.
- `command === "reset-timer"`일 때, **타이머 탭들을 찾아서** 리셋 신호를 보내야 함.
- 타이머 탭을 찾는 방법 — 두 가지 옵션, 아래 것을 권장:
  - (권장) `watcher.js`가 로드될 때 자기가 타이머 페이지(`#timerDisplay` 존재)라는 걸
    확인하면 `chrome.runtime.sendMessage({ type: "MAPLE_TIMER_PAGE_READY" })`를
    한 번 보내게 하고, background가 `Set`(`timerTabIds`)으로 관리.
    `chrome.tabs.onRemoved`에서 정리.
  - (대안) 매번 `chrome.tabs.query({})` 후 각 탭에 `executeScript`로
    `!!document.getElementById('timerDisplay')` 프로브 — 탭이 많으면 비효율적이라
    비권장.
- 대상 탭을 찾으면 각 탭에 `chrome.tabs.sendMessage(tabId, { type: "MAPLE_TIMER_HOTKEY_RESET" })`
  전송. (직접 `executeScript`로 페이지 함수를 호출하는 방식은 아래 5-3 이유로 비권장.)
- 이미 있는 `resolveTargetTabs`/`selectedTargets` 패턴과는 별개로 취급 권장.
  (미디어 정지 대상 탭 선택과 "리셋할 타이머 탭"은 성격이 다른 개념이라 섞으면 헷갈림.
  다만 여러 타이머 탭이 동시에 열려 있을 때 전부 리셋할지, 최근 활성 탭만 리셋할지는
  "열린 결정사항" 참고.)

### 5-3. `watcher.js` (content script — 브릿지 역할)
- content script와 페이지 스크립트는 격리된(world) 컨텍스트라 background가 직접
  페이지의 `fastRestart()` 함수를 호출할 수 없음. **CustomEvent로 다리를 놓아야 함.**
- `chrome.runtime.onMessage`에 리스너 추가:
  ```js
  chrome.runtime.onMessage.addListener((message) => {
    if (message?.type === "MAPLE_TIMER_HOTKEY_RESET") {
      document.dispatchEvent(new CustomEvent("MAPLE_TIMER_HOTKEY_RESET"));
    }
  });
  ```
- 기존처럼 `isTimerPage` 체크 안에서만 동작하게 두면 됨 (이미 그 가드가 있음).
- (5-2의 권장 옵션을 쓴다면) 페이지 로드 시 `MAPLE_TIMER_PAGE_READY` 전송도 여기서.

### 5-4. `index.html` (페이지 스크립트 쪽 — 이벤트 리스너 1줄 추가)
- 기존 `window.addEventListener('keydown', ...)` 블록 근처에 아래 추가:
  ```js
  document.addEventListener('MAPLE_TIMER_HOTKEY_RESET', () => {
    fastRestart(); // 스크롤/알림 클릭과 완전히 동일한 동작 재사용 (LOOT 적립 + 소리 + 재시작)
  });
  ```
- `fastRestart()`는 내부에서 `core.start()` → `notifyReset()`을 이미 호출하므로,
  기존 `data-reset-seq` → `watcher.js` 감지 → 미디어 재생 재개 파이프라인이
  **그대로 자동으로 연결됨.** 추가 배선 불필요.
- PiP 창(`pipWindow`)에도 동일 로직이 필요한지는 열린 결정사항 참고.

### 5-5. `popup.html` / `popup.js` (설정 UI)
- 새 섹션 "리셋 단축키" 추가. 구성 예:
  - 현재 등록된 단축키 표시 (`chrome.commands.getAll()`로 조회해서 `reset-timer`
    커맨드의 `shortcut` 필드 출력. 비어있으면 "설정 안 됨" 표시).
  - "단축키 변경" 버튼 → `chrome.commands.openShortcutSettings({ name: "reset-timer" })`
    (지원 안 하면 `chrome://extensions/shortcuts` 안내 문구 + 클립보드 복사 버튼으로 대체).
  - 안내 문구(제약사항 1번 내용을 사용자용 문구로): "실제 키 조합 변경은 크롬
    단축키 설정 화면에서 이루어집니다."
  - (선택) "테스트" 버튼 — 팝업에서 눌렀을 때 background로 같은 메시지를 보내서
    실제 리셋이 되는지 확인할 수 있게. `sendTestNotification`처럼 기존 프로젝트에
    이미 있는 "테스트" 패턴과 일관되게 만들면 자연스러움.

## 6. 열린 결정사항 (다음 Claude가 사용자와 확인 후 확정)

- [ ] 타이머 탭이 여러 개 열려 있으면: 전부 리셋? 마지막으로 활성화됐던 탭만 리셋?
      (기존 미디어 제어는 "선택 안 하면 전체 대상"이 기본값이었음 — 일관성 고려)
- [ ] PiP 창에서도 단축키 리셋이 필요한가, 아니면 메인 탭만으로 충분한가?
- [ ] 기본 제안 단축키를 `Ctrl+Shift+9`로 할지, 다른 숫자로 할지 (메이플 실사용
      테스트 후 확정 권장)
- [ ] 스크롤 리셋 기능은 유지할지(둘 다 지원), 단축키로 완전 대체할지

## 7. 정책 관련 메모 (참고용, 이전 대화 요약)

- 이 기능은 게임에 어떤 입력도 주입하지 않고, Chrome이 공식 제공하는 `chrome.commands`
  API로 단축키를 감지만 함 → 매크로/핵 정의에 해당하지 않는다는 이전 판단이 그대로 유지됨.
- 오히려 별도 네이티브 후킹 프로그램이 필요 없어졌으므로, "메이플과 함께 실행되는
  비정상 프로그램" 카테고리로 오인될 리스크도 이전 설계보다 낮아짐.
- 그래도 100% 공식 확답은 아니므로, 배포/공유 규모가 커지면 넥슨 고객센터 문의를
  권장한다는 기존 결론은 유지.
